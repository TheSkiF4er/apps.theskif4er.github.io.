Подход

Витрина должна показывать систему, а не объяснять её длинными абзацами.

Этот лендинг собран в том же тёмном минималистичном контуре, что и основной бренд: холодный акцент, строгая сетка, короткие формулировки и инженерный тон. Всё рассчитано на GitHub Pages и не требует сборки.

01 / product first

Сначала рабочее состояние

В демо-контуре пользователь должен увидеть продукт за секунды: интерфейс, состояние, ссылку на код и понятный маршрут к документации.

02 / traceable

Прозрачный переход в код

Каждая карточка ведёт не в пустой маркетинговый слой, а в обзор, репозиторий и docs. Демо должно быть проверяемым и технически прослеживаемым.

03 / static-first

Без лишней инфраструктуры

Чистый HTML, CSS и JavaScript. Никаких тяжёлых фреймворков и build-step, который усложняет публикацию на GitHub Pages.

Сигнал
Демо должно открывать продукт за секунды, а не убеждать пользователя, что продукт существует.

Редактирование

Витрина управляется через один файл.

Каталог карточек вынесен в app.js. Достаточно заменить названия, описания, теги и ссылки, чтобы перевести текущую витрину из обзорного режима в live-demo режим без правки разметки.

Витрина

Одна сетка для всех направлений: движки, CMS, экосистемы и лаборатории.

Ниже — стартовый контур из 0 карточек на базе текущей экосистемы SkiF4er. Сейчас карточки открывают обзоры, GitHub и docs. Когда появятся live-demo URL, достаточно заменить primaryUrl в app.js.

Подход намеренно прагматичный: даже если live-demo ещё не развернуто, карточка всё равно должна быть полезной и вести в код, документацию или обзор продукта.
Поток

Каждая демо-точка должна вести пользователя в инженерный контур за несколько шагов.

Витрина не заканчивается на красивой карточке. Её задача — быстро довести человека до исходников, документации и обратной связи.

Шаг 01

Открыть демо или обзор

Первая кнопка всегда отвечает за быстрый вход: live-demo, обзор продукта или страница системы.

Шаг 02

Сразу проверить код

Вторая кнопка должна вести в GitHub без обходных маршрутов и скрытых лендингов.

Шаг 03

Понять контекст через docs

Третья кнопка — это документация, заметки по архитектуре или точка входа в экосистему.

Шаг 04

Вернуться с фидбеком

После просмотра человек должен легко выйти в Telegram, Discord или GitHub issues.

Контур

Одна витрина. Три точки входа. Ноль лишнего шума.

Даже между релизами сайт остаётся полезным: обзор продукта, кодовая база и docs работают как единая поверхность для тех, кто хочет посмотреть систему в деле.
